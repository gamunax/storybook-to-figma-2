## New Project Setup
Please note, this is a `main-fe` only seed. Our full stack seed is on the way!

```bash
  # clone our repo
  git clone git@bitbucket.org:halopowered/hdf-ui-seed-main-fe.git name-of-project

  # change directory to our repo
  cd name-of-project

  # install the repo with npm
  yarn

  # start the server
  yarn start
```

## Custom Package Setup
If you're not using our seed repository, here's the basic package and module setup:
1. Install the desired HDF packages:
  ```bash
    npm install @halo/cdk@^3.0.0
    ```
2. Use the modules in your Angular application:
    
    ```jsx
    import { CdkModule } from '@halo/cdk';
    ```

> ❗️ Dependency
> 
> Please note, the `atlas-ui-field2` package is dependent on `luxon`. <a href="?path=/story/adopters-components-field-readme--page">More info</a>

## CDK Assets

System level assets are available for use in the project and are stored in the CDK. In this directory, you will find all of the basic svg, logos, docs and images that can be used by components. Before consuming the assets, it is necessary to set up the path into the project's main build. 

Add this configuration into `angular.json`:

```
  "assets": [
    {
      "glob":"**/*",
      "input": " ./node_modules/@halo/cdk/assets/logos",
      "output": "atlas-logos"
    },
    {
      "glob":"**/*",
      "input": " ./node_modules/@halo/cdk/assets/icons",
      "output": "atlas-icons"
    },
    {
      "glob":"**/*",
      "input": " ./node_modules/@halo/cdk/assets/docs",
      "output": "atlas-docs"
    },
    {
      "glob":"**/*",
      "input": " ./node_modules/@halo/cdk/assets/images",
      "output": "atlas-images"
    }
  ]
```
